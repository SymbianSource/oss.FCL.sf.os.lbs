<p>Note that the network status is only checked when LBS requests a Protocol Module to start a MOLR. The LBS subsystem would not attempt to stop positioning if network coverage was lost (either temporally or permanently) during an ongoing MOLR session.<br/></p><p>In such a case the following scenarios are possible:<br/></p><p>   - the AGPS Module generates an accurate or inaccurate location update despite the lack<br/></p><p>     of assistance data (Autonomous or Prefer Terminal Based modes)<br/></p><p>   -  the request times out (Always Terminal Assisted or Prefer Terminal Assisted modes)<br/></p><p>   - the Protocol Module decides to terminate the MOLR and the AGPS Module calculates position<br/></p><p>     autonomously (Prefer Terminal Assisted mode only)<br/></p>
